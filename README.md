## 拘ったポイント工夫したポイント

- ダークモードとライトモードを実装しました。
  - システム依存にし、アイコンクリックでダークモードとライトモードを切り替えられるようにしました。
  - レンダリングの関係でハイドリューションエラーが出るのでハイドリューションが完了して表示するように`Skeleton`を表示するようにしました。
- 一覧画面では10件取得するようにし、スクロールでページングするようにしました。
  - 取得した件数をそのまま使用してページネーションなどを行うとAPI rate limitを超えてしまい解決するには認証情報を使用する必要があります。
  - 認証情報を`.env`に入れて公開もできないので今回は最大件数を10件に制限しました。
- API実行中はローディングアニメーションを表示するようにしました。
- クライアントバリデーションの実装
  - React Hook Formで状態管理している項目を対象にバリデーションを実装しました。
  - 未入力で検索を行うとエラーが起きるのでクライアントバリデーションを実装しました。
  - 一部の特殊文字を使用できないようにバリデーションを実装しました。
  - 最大文字数を超えている場合はエラーが起きるので最大文字数バリデーションを実装しました。
- サーバーサイドバリデーションの実装
  - API RouteもURLを直入力して実行できてしまうのでバリデーションを実装しました。
  - 未入力で検索を行うとエラーが起きるので未入力バリデーションを実装しました。
  - 一部の特殊文字を使用できないようにバリデーションを実装しました。
  - 最大文字数を超えている場合はエラーが起きるので最大文字数バリデーションを実装しました。
  - ハイフンで始まるまたはハイフンで終わる場合はエラーが起きるのでハイフンバリデーションを実装しました。
  - ページが数字で入力されていない場合はエラーが起きるのでページバリデーションを実装しました。
- 詳細画面は激しい変更もないのでJSXを含めてキャッシュを行うようにしました。(5分間はキャッシュを使用する)
- 詳細画面では画面遷移後からコンテンツ表示若干のタイムラグがあるのでローディングアニメーションを表示するようにしました。
  - アバター画像もコンテンツ表示後も少し待って表示されるので別途アニメーションを表示するようにしました。
- 言語はテキストを表示するだけでは味気なかったのでタグっぽく表示するようにしてみました。
- Star数などはテキストを表示するだけでは味気なかったのでアイコンとセットで表示するようにしてみました。
  - 後々表示するコンテンツが増えても良いようにGridレイアウトで実装しました。
- 詳細情報取得時にエラーが発生した場合はエラーテキストと検索画面に戻るボタンを表示するようにしました。
- 検索から詳細に遷移し、戻るボタン、ブラウザバックした時は検索結果を保持するようにしました。
- 詳細からタイトルクリックで検索画面に戻るときはページングがリセットされるようにしました。
- 検索画面でリポジトリーが見つからない、エラーが発生した場合はエラーテキストを表示するよう対応しました。
- 不明なURLにアクセスした場合は404エラー画面を表示するよう対応しました。
- 入力フォームとボタンもハイドリューションが変わる前に入力できており、完了後にリセットされるのでハイドリューションが完了するまではSkeletonを表示するようにしました。
- レスポンシブ対応を行いました。
- `pnpm-workspace.yaml`を使用して`minimumReleaseAge`を設定してサプライチェーン攻撃の考慮を行いました。
- `.npmrc`を使用して`ignore-scripts`を設定してサプライチェーン攻撃の考慮を行いました。
- アクセシビリティ対応を行いました。
- ユニットテストとe2eテストの両方を行いました。

## AIを使ったポイント

- Emotion依存のHydrationErrorがドキュメント通りにしても直らなかったのでAIに質問しました。
- ダークモードとライトモードの切り替えアイコンでHydrationErrorが発生したので解決方法がないか質問しました。
- 各タスク完了時に事前に設計したrulesなどを元にレビューしてもらいました。(修正してもらうのではなくあくまで指摘してもらう)
- テスト項目に不備がないか相談。
- GitHub APIの仕様について質問。
- `TopContent.tsx`内の処理が冗長になっていたのでリファクタリングできないか相談。
- `TopContent.tsx`内ので当初は`useForm`から参照した`isSubmitting`を使用していたが、ページネーションで遷移する時もdisabledの有効やアニメーションの表示を行いたいが、`submit`しないので`isSubmitting`では網羅できなかったので代替え案の相談。
  - AIに相談したところ、`useState`を使用した状態管理を提案したが冗長な状態管理と判断し、提案を却下した上で`useTransition`を使用した状態管理を採用。
- `RepositorySearchForm`で使用しているYamadaUIのコンポーネントや関数のモック化の方法を質問。
- クエリパラーメーターを受け取るサーバーコンポーネントをテストコードでレンダリングする方法を質問。
- e2eテストでキャッシュコンポーネント起因のエラーが起きたのでどう対処するか質問。
- e2eテストのテスト内容に不備が無いか相談。
- 当初e2eテストはVitestのブラウザモードを想定していたがブラウザモードはjsdomではなくブラウザを使用したユニット、コンポーネントテストを行う機能
  - e2eテストをPlaywrightを使用して行うよう修正したので必要なくなったスクリプトやパッケージなどを洗い出し
- firefoxでe2eテストが動かなかったので対処方法を質問。
- 検索結果を保存する為の設計の相談(クエリーパラメーターを使用するか、localStorageを使用するか)
  - 検索処理の実行タイミングの相談
  - Server ActionからAPI Routesを使用してキャッシュする方法で解決
- 検索結果のリストのレスポンシブがレイアウト崩れを起こしていたので対応方法の相談。
- 検索結果表示後のバリデーションが正しく動いていなかったので対応方法の相談。
  - React Hook Form のバリデーション自体は実行されていたが、エラー状態の表示を `Form.Root` 経由で間接的に行なっていた為今回の画面構成では UI に反映されなかった。そのため、`Field.Root` に `invalid` と `errorMessage` を直接渡すように変更した。
- GitHub API側のバリデーション仕様について質問。
